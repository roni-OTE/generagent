03 ARIA patterns · do / don't

שלושת הדפוסים הנפוצים — מה נכון, מה לא.

icon button · bad
no label
<button> <svg>...</svg> </button> // screen reader: "button" // (no idea what it does)
icon button · good
aria-label
<button aria-label="סגור מודאל"> <svg aria-hidden="true">...</svg> </button> // screen reader: "סגור מודאל, button"
loading state · bad
silent
<div className="orb breathing"/> <p>analyzing...</p> // screen reader: ignores // status changes silently
loading state · good
live region
<div role="status" aria-live="polite"> <p>analyzing</p> </div> // screen reader: "analyzing" // announced when changes
checkbox · bad
div soup
<div onClick="toggle"> <div className="box"/> <span>אני מסכים</span> </div> // not focusable, no Tab, no Space
checkbox · good
native input
<label> <input type="checkbox"/> <span>אני מסכים</span> </label> // Tab + Space, accessible by default

05 RTL · 7 כללים

הדברים שמעצבים שוכחים כשהם מעבירים מאנגלית לעברית.

01
השתמש ב-logical properties
במקום margin-left השתמש ב-margin-inline-start. אוטומטית מתהפך ב-RTL.
❌ margin-left: 16px;
✓ margin-inline-start: 16px;
02
flex gap עובד אוטומטית
display: flex; gap: 16px; — אין צורך לטפל ב-RTL ידנית. ה-spacing נכון בשני הכיוונים. אם השתמשת ב-margin-right במקום gap — תצטרך לתקן.
03
חצים וקוד תמיד LTR
חץ → במשפט עברי צריך להראות "מימין לשמאל". בקוד / מספרים / IDs — תמיד עטוף ב-<span dir="ltr">.
✓ <code dir="ltr">npx generagent install</code>
04
SVG אייקונים — שני כיוונים
חץ שמאל ב-LTR = חץ ימין ב-RTL. השתמש ב-transform: scaleX(-1) לחיצים, או SVG נפרדים. לוגואים לא הופכים.
05
תאריכים ומספרים
תאריכים בעברית: "8 ביוני 2026" (כיוון ימין-שמאל הוא טבעי). מספרים גדולים: "1,234" — תמיד עם פסיק כמפריד (לא נקודה).
06
Font fallback ל-mixed content
font-family: 'IBM Plex Sans Hebrew', 'Inter', system-ui — האותיות העבריות מ-Hebrew, הלטיניות מ-Inter. שניהם נראים מאוזנים.
07
scroll-margin ולא margin
לעוגנים (#section) — השתמש ב-scroll-margin-top כדי שכותרות לא יסתתרו תחת nav. זה לא תלוי כיוון.

07 סיכום · do / don't

✓ DO
  • השתמש ב-semantic HTML — <button>, <label>, <nav> במקום divs
  • תן aria-label לאייקונים — כפתורי × ⌘ ⋯ חייבים שם נגיש
  • שמור focus-visible — לעולם אל תכבה את ה-outline בלי תחליף
  • בדוק עם VoiceOver — Cmd+F5 על Mac, נסה לנווט עיוור
  • הוסף role="status" / aria-live לכל תוכן דינמי (toasts, bot replies)
  • השתמש ב-logical properties — margin-inline-start במקום margin-left
✗ DON'T
  • אל תשתמש ב-color בלבד ל-status — תמיד צריך דוט / אייקון / טקסט
  • אל תשתמש ב-placeholder כ-label — נעלם כשמקלידים
  • אל תבטל animations ב-reduced-motion — תאט או החלף
  • אל תוסיף tabindex חיובי — מבלגן את Tab order הטבעי
  • אל תשים אצבע מתחת ל-44px — touch targets קטנים = שגיאות מובייל
  • אל תשכח dir="ltr" על קוד / URLs / IDs בתוך טקסט עברי
