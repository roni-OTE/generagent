הבעיה והפתרון

למה צריך את GenerAgent בכלל?

⚠️
הבעיה
קל ליצור סוכן AI, אבל קשה להבין איזה סוכן באמת צריך. רוב המשתמשים נכנסים ל-Claude Code עם דף ריק ולא יודעים מה לבנות — לא מבינים MCP, לא יודעים לחבר OAuth, ולא מבינים מה ההבדל בין skill לבין agent.
✨
הפתרון
בוט שיחה שמראיין את המשתמש כיועץ עסקי מקצועי — שואל על התפקיד, סדר היום, ותחומי האחריות. בסוף, מפיק חבילת ZIP מוכנה עם כל הקבצים והוראות התקנה צעד-אחר-צעד בעברית פשוטה.
🎯 הצעת הערך בשורה אחת
"ספר לי על העבודה שלך — אני אכין לך את הסוכן המושלם."

הבוט — מנוע שיחה אדפטיבי

לא שאלון קבוע. הבוט מנחה את המשתמש לגלות את הצורך שלו — תוך הסתעפות לפי התשובות וגבולות קשים

💡 פילוסופיה — סוקרטי, לא דריל
רוב המשתמשים לא יודעים מראש איזה סוכן הם צריכים. הבוט שואל שאלות פתוחות ומסבירה למה הוא שואל ("שאלה זו תעזור לי להבין X"), ובוחר את השאלה הבאה בהתאם למה שלמד עד עכשיו. 7-15 שאלות בלבד, עם early-exit כשהביטחון גבוה.

שלושת השלבים

🔍
Phase 1 — Discovery
3-4 שאלות פתוחות תמיד נשאלות. המטרה: לבנות תמונה כללית של המשתמש ולזהות persona ראשוני.
"ספר לי בכמה משפטים — מה אתה עושה ביום-יום?"
"אם הייתה לך עוד שעה ביום, מה היית עושה איתה?"
"אילו אפליקציות אתה פותח הכי הרבה?"
🌳
Phase 2 — Deep Dive
3-7 שאלות מסתעפות נבחרות דינמית מבנק שאלות בהתאם ל-persona שזוהה.
Knowledge Worker:
"כמה אחוז מהמיילים שלך לא רלוונטיים?"

Manager:
"איך אתה עוקב אחרי משימות הצוות?"
🎯
Phase 3 — Refinement
1-4 שאלות גימור. כאן הבוט כבר מציג המלצה ראשונית ושואל שאלות מכוונות לחידוד.
"מתוך 3 הארכיטיפים האלה — איזה מרגיש לך הכי קרוב? וכדי לדייק, ספר לי..."

דוגמה ל-Branching

consultation-flow.txt
[Discovery] Bot: "ספר לי בכמה משפטים — מה אתה עושה ביום-יום?" User: "אני רואה חשבון עצמאי, מנהל ~40 לקוחות" Bot: "אם הייתה לך עוד שעה ביום, מה היית עושה איתה?" User: "הייתי משיב למיילים בזמן... אני מפגר שבוע" Bot: "אילו אפליקציות אתה פותח הכי הרבה?" User: "Gmail, Calendar, ו-Excel" # Claude מנתח: persona=knowledge-worker-solo, confidence=0.72 # נדרש Deep Dive ב-2 כיוונים: email overload + scheduling [Deep Dive — branched to "email overload"] Bot: "מתוך המיילים שאתה מקבל ביום, איזה אחוז דורש תגובה ממשית?" → שאלה זו תעזור לי להבין אם הסוכן צריך לסנן או לסכם User: "אולי 30%, השאר זה הודעות מערכת" Bot: "כשמגיע מייל חשוב מלקוח — איך אתה יודע שהוא חשוב?" User: "לפי השולח. יש לי 5 לקוחות VIP" # confidence=0.91 → EARLY EXIT לשלב הבא (פילגתי 5 שאלות במקום 8) [Refinement] Bot: "על סמך מה שספרת, אני חושב שתועיל לך אחת מ-2 חבילות: [A] Email Triage Assistant — מסנן ומסכם [B] Client-Focused Inbox — VIP routing + priority איזו מרגישה יותר מתאימה?" User: "B" ✓ סיום אחרי 7 שאלות (במקום 12 קבועות) ✓ ביטחון סופי: 0.94

גבולות קשים — אין לופ אינסופי

Guardrail
ערך / מטרה
MAX_QUESTIONS
15 — תקרה קשה. מעבר לזה הבוט סוגר עם הנתונים שיש
MIN_QUESTIONS
7 — לא מפסיק לפני, אחרת ההמלצה רדודה
EARLY_EXIT_CONFIDENCE
0.85 — אם הביטחון גבוה, דלג ישר ל-Refinement
MAX_TIME_MINUTES
8 — אם המשתמש איטי, מציע "המשך מאוחר" עם שמירת state
MAX_RETRIES_PER_Q
2 — אם התשובה לא ברורה, מנסה ניסוח אחר ואז עובר הלאה
FALLBACK_ARCHETYPE
general-assistant — אם הביטחון נשאר נמוך → ברירת מחדל בטוחה

זרימת המנוע

1
Discovery (3-4 שאלות פתוחות)
איסוף הקשר → זיהוי persona ראשוני ע"י Claude
2
Persona Detection (אוטומטי, ללא שאלה)
Claude מחזיר confidence + 1-3 archetypes חשודים
3
Deep Dive (3-7 שאלות מסתעפות)
שאלות נבחרות דינמית מבנק לפי persona + early-exit אם confidence ≥ 0.85
4
Tentative Recommendation
הבוט מציג 2-3 ארכיטיפים אפשריים — לא רק רשימה אלא הצעה
5
Refinement (1-4 שאלות גימור)
חידוד + טכניות + רגישות נתונים + אישור
6
→ Build Package
סה"כ 7-15 שאלות (ממוצע צפוי: 9)

מה המשתמש מקבל בסוף התהליך

📦
my-agent-package.zip
~50KB · מוכן להתקנה · עברית
my-agent-package.zip
my-agent-package/ │ ├── README.md # הסבר כללי בעברית ├── INSTALL.md # מדריך התקנה צעד-אחר-צעד ├── LEGAL.md # הסכם חתום + hash │ ├── .claude/ # עבור Claude Code │ ├── settings.json │ ├── agents/ │ │ └── personal-assistant.md │ └── skills/ │ ├── email-triage/ │ │ └── SKILL.md │ └── calendar-summary/ │ └── SKILL.md │ ├── codex/ # עבור Codex CLI │ ├── AGENTS.md │ └── codex.config.toml │ ├── mcp/ # MCP Connectors │ ├── mcp.json │ └── README.md # הוראות OAuth │ └── scripts/ ├── install.sh # Mac/Linux └── install.ps1 # Windows
📖
INSTALL.md מותאם
מדריך התקנה בעברית פשוטה — מותאם לרמת הטכניות שהמשתמש דיווח בבוט. רמה 1 = "פתח את היישום הזה ולחץ פה". רמה 5 = פקודות bash בלבד.
🔌
הוראות OAuth ויזואליות
אם הסוכן צריך Gmail או Calendar — מדריך מפורט עם screenshots placeholders: "פתח Gmail → Settings → Developer Console → Create Project..."
⚖️
LEGAL.md חתום
כל חבילה כוללת snapshot של ה-TOS שהמשתמש חתם עליו, IP, תאריך, וגרסה. Hash של החבילה לוודא שלא שונתה.
